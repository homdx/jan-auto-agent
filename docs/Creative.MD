1. Настроить agents.ini под creative
В секции [auto] переключить режим и проверить бюджеты:
ini[auto]
task_mode = creative          ; ← ключевое, режим читается отсюда, не из CLI

[coder]
num_ctx_creative    = 8192
max_tokens_creative = 2048
(остальные creative-ключи — max_compression_passes, canon_check_every и т.д. — уже на месте после CR-5/CR-7).
2. Подготовить папку книги
bashmkdir -p ~/book && cd ~/book

# Завязка — это и есть «начало героев». Без неё架итектору не на чем заземлиться.
cat > chapter_01.md <<'EOF'
# Глава 1

Капитан Рейес стоит на мостике «Альбатроса». Ей сорок,
седина на висках, шрам над левой бровью. Рядом — юнга Мира,
впервые в открытом море. Корабль идёт на север, к Ледяным
проливам, за грузом, о котором команде знать не положено.
EOF

# Целевую главу создаём ЗАРАНЕЕ пустой/заглушкой — это target,
# который coder заполнит (в creative target обычно новый/пустой файл).
: > chapter_02.md
Почему chapter_02.md создаём руками пустым: architect берёт target_files из того, что нашёл ингест. Чтобы он гарантированно «прицелился» в нужную главу и заземлился на chapter_01.md, проще дать ему оба файла.
3. Запуск
bashcd ~/путь/к/проекту_с_main.py

python main.py \
  --auto "Напиши главу 2, продолжая историю из главы 1" \
  --base ~/book \
  --config agents.ini
Эквивалент из интерактивной оболочки:
bashpython main.py --base ~/book
# затем внутри:
/auto Напиши главу 2, продолжая историю из главы 1
Сухой прогон без записи (посмотреть план до генерации):
bashpython main.py --auto "Напиши главу 2..." --base ~/book --dry-run
4. Что появится в папке после прогона

chapter_02.md — заполнена прозой.
synopsis.md — создаётся автоматически после принятия главы (CR-5): по секции на главу, это и есть «краткая версия», от которой пишется дальше.
.agent/ — состояние прогона (план, прогресс, метрики), resume берётся отсюда.
git-репозиторий инициализируется автоматически (_setup_git), каждая принятая глава коммитится.

5. Следующие главы — тот же цикл
bashcd ~/book && : > chapter_03.md          # пустой target для главы 3
cd ~/путь/к/проекту && python main.py \
  --auto "Напиши главу 3, продолжая сюжет" --base ~/book --config agents.ini
На главе 3 (кратна canon_check_every=3) впервые отработает canon-гейт: сверит факты главы 3 с synopsis.md + главами 1–2 и, если, например, Рейес вдруг «погибла, но действует» — вернёт на переписывание (один раз, потом примет с warning).
